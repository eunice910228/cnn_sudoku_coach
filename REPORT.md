## 從感知到推理:以數獨為例的神經符號系統實作
*A Neuro-Symbolic Approach: from Perception to Reasoning*

### Slide 2|破題:「丟給 ChatGPT 不就好了?」
📌 主動把全場最想問的問題端上桌,順便框出「白盒 vs 機率」的主軸。
🎤 「LLM 是機率模型:它能生出*聽起來*有道理的解釋,但那是事後編的,可能跟真正的計算無關,甚至自信地出錯。本系統的推理是白盒——理由不是『講出來的』,是『算出來的』,每一步可驗證、保證正確。數獨只是載具,真正想示範的原則是:**有精確解的問題,別用機率的鎚子敲。**」

### Slide 3|系統總覽:兩個半腦(架構背景)
📌 架構圖(README 有):感知模組(CNN,會看不會講理)→ 81 個數字 →推理模組(技巧引擎,會講理不會看)。一句話帶過「為什麼這樣分」,把舞台讓給後面的 CNN。
🎤 「神經網路會看不會講理、符號邏輯會講理不會看,剛好互補——這個組合叫 Neuro-Symbolic。**但這份報告的主角是前半段那顆 CNN:它到底學到什麼、什麼時候會錯、錯了怎麼辦。**」

### Slide 4|感知 I:前處理 + 為什麼不用 MNIST
📌 流程圖:灰階 → 自適應二值化 → 最大輪廓(凸四邊形 epsilon 掃描)→ 透視校正 → 切 81 格(`vision/grid_extractor.py`)。並點出**不用 MNIST**:數獨是印刷體、MNIST 是手寫,分布不同 → 用字型渲染合成資料(`vision/digit_dataset.py`)。
⚠️ 自備:自己拍一張數獨,放前處理的 before / after 截圖。
🎤 「訓練/測試分布要一致,模型才有效——所以資料是用字型『印』出來的,不是借手寫的 MNIST。」

### Slide 5|感知 II:CNN 架構(逐層形狀)
📌 `vision/digit_cnn.py`:Conv(32,3×3)→Pool→Conv(64,3×3)→Pool→Flatten→Dense(128)→Dropout→Softmax(10)。重點講**形狀怎麼變**:28→26→13→11→5→1600→128→10,以及「前半段抽特徵、後半段做決定」。
🎤 「卷積在學一組濾波器、池化給平移容忍、全連接做分類;參數大多在 Dense,所以 Conv 很省卻很會看。」(想講得更直觀,可現場開 `learn/lab.py` 的手算卷積)

### Slide 6|CNN 學到什麼:濾波器 + 特徵圖
📌 ⚠️ 自備:`experiments/visualize_cnn.py` 產出的 `filters.png`(第一層 32 個濾波器)、`feature_maps.png`(一個真實數字觸發了哪些通道)。
🎤 「第一層自己長出了邊緣/筆畫偵測器——沒有人教它,是從資料裡長出來的。」

### Slide 7|CNN 在「看哪裡」:Grad-CAM
📌 ⚠️ 自備:`gradcam.png`——熱力圖疊在數字上,顯示模型判斷每個數字時注意力落在哪塊像素。
🎤 「這回答了『黑盒到底看哪裡』:紅色是關鍵區。可以看到它確實盯著筆畫,不是在看背景——這讓辨識結果更可信。」

### Slide 8|訓練與評估:混淆矩陣 + 特徵分群
📌 ⚠️ 自備(全頁):`vision/train.py` 印出的測試準確率與**混淆矩陣**;加 `experiments/extras.py` 的 **t-SNE**(`tsne.png`,倒數第二層特徵把 1~9 與空格分成 10 群)。指出最易混淆的數字對,講一句為什麼。
🎤 「t-SNE 證明模型學到的是『可分離的語意表徵』,不是死背;混淆矩陣告訴我們它最容易在哪裡犯錯。評估講得出道理,比準確率高低更加分。」

### Slide 9|⭐ Sim-to-Real:模型的真實極限在哪
📌 ⚠️ 自備:`experiments/eval_real.py` + `robustness.py`。核心發現——乾淨數位輸入幾乎 100%、合成測試 99.75%,**幾乎無落差**;但壓力測試(`robustness_curves.png`)顯示:對焦模糊、感光雜訊、JPEG 壓縮都很穩,**唯一弱點是光線不均**(準確率 100% → 76% → 69%)。
🎤 「我沒有硬湊『合成→真實一定掉很多』的故事——乾淨圖它就是 100%。於是我改用壓力測試找它真正的極限,結果是**光線不均**。這正好解釋了拍照提示為什麼寫『光線均勻』,也指出改進方向:前處理做光照校正、或在增強裡加入光照梯度。」

### Slide 10|⭐ 資料增強消融(Ablation)
📌 ⚠️ 自備:`experiments/ablation_aug.py`(`ablation_bar.png`、`ablation_robustness.png`)。訓練「有/無增強」兩個同結構模型對照:真實照片 **100% vs 87.65%(+12pp)**、帶擾動合成測試 **98.65% vs 67.20%(+31pp)**。
🎤 「消融實驗 = 把一個設計拿掉、其餘不變,用數字證明它的價值。資料增強不是裝飾,它買到了 12 個百分點的真實穩健性。」

### Slide 11|⭐ 實戰除錯:白盒抓到黑盒的錯,並修在源頭
📌 ⚠️ 自備(故事頁,放截圖):真實照片把**左上角空格**誤判成數字 → 推理引擎解到一半回報**「無解」**(合法數獨一定有解 → 代表辨識錯了)→ 診斷出根因是**空格殘留外框亮線**(訓練時的空格是純黑,模型沒看過)→ 修法:**只給空格類別**加「邊框殘留增強」(`digit_dataset._add_grid_residue`)→ retrain 後該情境 94.5%→~100%,且乾淨照片仍 100%、角落不再幻覺數字。
🎤 「這頁是三件事一次發生:① 白盒替黑盒把關的安全網**真的作動**;② 一個能複現的失敗;③ 用**針對性的資料增強**把它修在源頭——而且我證明了『增強只能加在空格,連數字也加會讓模型被邊緣帶偏、反而讀不出淡的數字』。這才是理解一個 CNN,而不只是會用。」

### Slide 12|推理模組:白盒技巧引擎(架構另一半)
📌 四招階梯:唯一候選數(★)→ 隱性唯一數(★★)→ 顯性數對(★★★)→ 區塊摒除(★★★☆);每步附中文理由;**難度評級 = 解開此題所需的最難技巧**。超出範圍時回溯法備援並誠實標記。
🎤 「這就是白盒:每一步有名字、有理由、可驗證——和前面那顆會看不會講的 CNN 形成對照。」

### Slide 13|教練設計 + Live Demo(雙 Demo,App 為主)
📌 系統靈魂 + 現場演示。**主角:桌面 App `python app/play_gui.py`**——拍照→辨識→填進格子(可疑格標橘);玩家**自己填、任何順序**,卡住按「下一步提示」,程式**讀『目前盤面』即時算出下一步思路**,新招還附心法;一次只給一步、不劇透。**備援:`python play.py`(CLI、零依賴)**證明引擎不綁框架。
🎤 「關鍵設計:提示是**從你當下的盤面**算出來的,所以你用什麼順序填都對得上——不存在『思路對不齊』的問題。目標不是把數獨解掉,是讓玩家自己變強;AI 站在玩家旁邊,不是對面。」
⚠️ 自備:事先演練一次 play_gui;準備截圖當 Plan B(現場 demo 永遠要有備案)。

### Slide 14|限制與展望(誠實)
📌 誠實列:**域偏移泛化不穩**——合成字型→某張真實照片的準確率有 run-to-run 差異,需靠 `restore_best.py` 多訓幾個挑最穩的(這本身是個值得談的 ML 現象);手寫數字尚未支援;進階技巧(X-Wing 等)未實作;未來:鏡頭常開的 AR 即時版(只認『變化的那一格』)、手機 App。也提**工程取捨**:辨識(重、需 TF)與遊玩(輕、零依賴)刻意解耦,讓玩的人不必裝 TensorFlow。
🎤 「會看的代價很高,所以我讓它只在開局出場一次;之後狀態存在程式裡,提示是微秒級。」

### Slide 15|結論
🎤 「神經網路會看、符號邏輯會想,讓它們各司其職。但這份報告真正做的,是**把那顆 CNN 拆開看**:它學到邊緣與筆畫、注意力落在筆畫上、在乾淨圖近乎完美、唯一弱點是光線、增強值 12 個百分點、空格殘留會害它幻覺——而我把那個幻覺修在了源頭。有精確解的問題,別用機率的鎚子敲;但就算用了機率的工具,也要量得出它的能力邊界。謝謝大家。」

## Q&A 預備彈藥(先想好,台上不會慌)

| 可能的提問 | 回答方向 |
|---|---|
| 為什麼不用 MNIST? | 分布不同:數獨是印刷體、MNIST 是手寫;訓練/測試分布一致才有效。 |
| 為什麼不讓 CNN 直接學「解數獨」? | 解題是硬約束的精確問題,神經網路無法保證正確且無法解釋;感知歸神經、推理歸符號。 |
| 單格準確率為什麼要那麼高? | 整盤 81 格全對機率約 p^81;單格 99% 也只剩約 44% 整盤對——所以才需要白盒矛盾偵測當安全網。 |
| 空格怎麼會被讀成數字?怎麼修的? | 真實角落空格殘留外框亮線,訓練的空格是純黑、沒看過 → 誤判;修法是**只給空格類別**加邊框殘留增強,retrain 後解決。 |
| 增強為什麼只加空格、不加數字? | 實測:連數字類別也加,模型會被邊緣亮線帶偏,反而讀不出對比較淡的真實數字(01.jpg 數字準確率直接崩)。 |
| 域偏移為什麼會「不穩」? | 合成→真實是分布外泛化,對訓練隨機性敏感,同設定 run-to-run 會差;用 `restore_best.py` 訓多個挑對真實照片最穩的。 |
| 技巧引擎和回溯法差在哪? | 技巧引擎模擬人類推理、每步可解釋;回溯法是試誤搜尋、保證找到解但給不出人看得懂的理由。 |
| 難度評級的依據? | 解開此題所需的「最難技巧」等級,可再加步數加權。 |
| 提示會不會因為填的順序而對不上? | 不會。提示用 `peek_next(目前盤面)` 即時重算,從你當下的狀態給下一步,順序無關。 |
