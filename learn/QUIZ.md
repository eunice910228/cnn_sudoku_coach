# 參考答案(先自己想過再看)

1. **一個 28×28 的數字矩陣**。CNN 不會「看見」數字,它只對這堆數字做數學運算。
2. **0 = 黑(沒筆畫)、1 = 白(有筆畫)**,中間是灰階。亮度被正規化到 0~1。
3. 把濾波器蓋住的那一小塊,和濾波器**對應位置相乘**,再把結果**全部加總**成一個數(加權和)。
4. 因為兩個濾波器的權重排列不同:垂直邊濾波器對「左右明暗差」反應強,所以點亮**直/斜的豎筆**;水平邊濾波器對「上下明暗差」反應強,所以點亮「7」**頂部的橫槓**。濾波器偵測什麼,取決於它的數字長怎樣。
5. 因為 3×3 濾波器的中心不能貼到最邊緣——左右各少 1、上下各少 1,所以 28→26。(這叫無填充 valid 卷積。)
6. **把所有負數變成 0**(留下有偵測到特徵的強反應,負的當沒看到);順帶讓網路能學非線性。
7. ① **尺寸變小** → 運算量下降;② **平移容忍** → 數字稍微移位,取最大值的結果幾乎不變。
8. **前半段 Conv/Pool 負責「抽特徵」**(從邊 → 部件 → 整體);**後半段 Flatten/Dense 負責「做決定」**(綜合所有特徵,判斷是哪個數字)。
9. 因為一個 Conv 濾波器只有很少權重(3×3)卻能**重複掃全圖共用**;而 Dense 是「每個輸入接到每個輸出」,連線數爆炸,參數自然多。
10. **猜**:看一批圖、輸出預測;**比**:跟正解比較,算出錯多少(loss);**調**:用梯度下降把所有權重往「錯更少」的方向挪一小步。重複幾千次。
11. loss 是「預測和正解差多少」的分數。我們希望它**變小**(越小=猜得越準)。
12. **訓練時自己學出來的**。一開始是隨機數,靠「猜→比→調」一輪輪調整,慢慢長成邊緣/筆畫偵測器。沒有人告訴它數字長怎樣。
13. **一份機率分布**(softmax):10 個類別各一個機率、加起來等於 1。取最大的那個當答案。
14. 那是 softmax 給「預測類別」的那個機率值。模型對「7」這格輸出的機率最高項是 0.987,所以說信心 98.7%。低信心 = 模型沒把握。
15. 因為訓練讓濾波器**自己長成了對邊緣/筆畫敏感的樣子**——這說明 CNN 的特徵不是工程師寫死的,是**從資料裡學出來的**。
16. **CNN 負責「看」**:把照片裡 81 格的數字辨識成 0~9(會看,但說不出理由);**推理引擎負責「想」**:用數獨規則一步步推、每步可解釋、保證正確。這個組合叫神經符號(Neuro-Symbolic)。
17. 靠**符號推理引擎做白盒把關**:CNN 若把某格認錯,會違反數獨規則(某格無解/同列重複),推理引擎一檢查就發現矛盾,回報輸入有誤。感知會犯錯,推理抓得到。
