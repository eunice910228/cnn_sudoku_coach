## 十分鐘心智模型:CNN 到底在幹嘛?

一句話:**CNN 是一台「自動學會抽特徵、再做分類」的機器。**

把它想成一條工廠生產線,一張圖從左邊進、一個答案從右邊出:

```
   圖片        抽特徵(會看)                     做決定(會分類)
  ┌────┐   ┌─────────────────────────┐      ┌──────────────────┐
  │ 7  │ → │ Conv → ReLU → Pool  ×N   │  →   │ Flatten → Dense  │ → 機率[0~9]
  └────┘   └─────────────────────────┘      └──────────────────┘
            「邊」→「部件」→「整體」            「綜合判斷是哪個數字」
```

四個核心動作,就這四個:

| 動作 | 白話 | 為什麼需要它 |
|---|---|---|
| **卷積 Convolution** | 小濾波器滑過全圖,做加權和,抓出某種特徵 | 一個濾波器掃全圖 = 同一個「特徵偵測器」到處找,**省參數又抓得到平移** |
| **ReLU** | 負數變 0 | 只留「有偵測到」的強反應;也讓網路能學非線性(複雜形狀) |
| **池化 Pooling** | 2×2 取最大,圖縮小 | 減少運算 + **數字稍微移位也不怕**(平移容忍) |
| **全連接 Dense** | 把所有特徵綜合起來投票 | 從「看到哪些特徵」做出「這是哪個數字」的最終決定 |

然後最關鍵的一句:**這些濾波器的數字不是人寫的,是「訓練」時自己學出來的。**
訓練 = 看大量例子 → 猜 → 對答案 → 算錯多少(loss)→ 微調參數讓錯變少 → 重複幾千次。

懂了這幾點,你就懂 CNN 了。`lab.py` 會把每一點變成你親眼看到的東西。

## 術語小抄(被問就能答)

- **濾波器 / 卷積核(filter / kernel)**:一個小矩陣(如 3×3),負責偵測一種局部特徵。
- **特徵圖(feature map)**:一個濾波器掃完整張圖後得到的「反應強度圖」。
- **通道(channel)**:有幾個濾波器,就有幾張特徵圖、幾個通道。第一層 32 個 → 32 通道。
- **參數 / 權重(parameter / weight)**:模型裡所有「可被訓練調整」的數字。本專案約 22.5 萬個。
- **epoch**:把整份訓練資料完整看過一輪。看 N 輪 = N 個 epoch。
- **loss(損失)**:衡量「預測和正解差多少」,越小越好。訓練就是想辦法讓它變小。
- **梯度下降(gradient descent)**:算出「往哪個方向調參數會讓 loss 變小」,然後挪一小步。
- **softmax**:把模型輸出變成「加起來等於 1 的機率分布」。
- **過擬合(overfitting)**:死背訓練題、考新題就垮。`Dropout` 就是用來防它的。

## 三個一定會被問(也值得想清楚)的問題

**Q：為什麼用卷積,不直接用一般的全連接神經網路?**
A：全連接會把每個像素都接到每個神經元,參數爆炸、又學不到「位置無關」的特徵。
卷積用「同一個小濾波器掃全圖」,參數少很多,而且數字出現在哪都抓得到。

**Q：CNN「學到」的東西到底是什麼?**
A：是那些濾波器裡的數字。訓練讓第一層長成邊緣/筆畫偵測器,深層再把它們組合成
數字的部件——跑第 7 關就能親眼看到。

**Q：它會出錯嗎?怎麼辦?**
A：會。輸出是機率,信心低就代表它沒把握。本專案的巧妙處:用**符號推理引擎**
當白盒,反過來抓 CNN 的錯(數獨規則一矛盾就知道某格認錯了)。見第 8 關。

## 一句話收尾

> CNN 不是魔法。它只是「卷積抽特徵 + 全連接做決定」,再用「看例子調參數」
> 把濾波器練出來。你手算過、訓練過、看過它學到什麼——
> 現在它對你來說,是一台**看得懂內部的機器**了。🎓
